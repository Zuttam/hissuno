# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Hissuno (the platform, widget, or CLI), please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email us at **security@hissuno.com** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours of your report
- **Assessment**: We will evaluate severity and provide an initial timeline for a fix
- **Fix**: Security patches are prioritized and released as soon as possible
- **Disclosure**: We will coordinate with you on public disclosure timing

## Scope

This policy covers:

- The Hissuno web application (`app/`)
- The embeddable widget (`@hissuno/widget`)
- The CLI (`hissuno`)
- API endpoints and authentication

## Out of Scope

- Vulnerabilities in third-party dependencies (report these to the upstream project)
- Issues in development/test configurations that do not affect production

Thank you for helping keep Hissuno secure.
